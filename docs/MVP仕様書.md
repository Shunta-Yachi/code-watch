# MVP仕様書

App Ver: v0.1.0

## 1. 概要

- 作業時間記録用のVS Code拡張機能(VSCE)
- 初期段階では個人利用を想定
  - 将来的にはチーム・プロジェクト単位での利用も視野

## 2. 技術スタック

- TypeScript
- SQLite

## 3. 機能

- メインテーマ: VS Codeで作業している時間を記録する

### 3-1. 作業時間計測

- 計測はオートスタート・オートストップ
  - スタート条件: VS Code Workspaceが開かれたとき
  - ストップ条件: VS Code Workspaceが閉じたとき

### 3-2. 作業時間記録

- 計測した作業時間を記録する
- 記録先はSQLiteを採用
  - 理由: 1ファイル完結の手軽さ、競合処理
  - テーブル設計は `docs/テーブル設計書.md` を参照
- 記録タイミング:
  - 1: 計測が終了したとき
  - 2: 計測中、定期的に保存(ハートビート)
    - デフォルト1分毎
- 複数ワークスペースが開かれている場合、同数のセッションを並行稼働させる

### 3-3. 非作業時間記録

- 以下のような動作が検出された場合、非作業時間としてカウントする
  - sleep: PCがスリープ状態だった場合
  - unfocused: VS Code ウィンドウにフォーカスがなかった場合
  - idle: VS Code ウィンドウにフォーカスはあるが、操作されていない場合
- ただし、これらはすべてDBに保存するとレコード数が肥大化し破綻する(特にidle)
  - そのため、一定の閾値を拡張機能側で設定し、その閾値を超えた場合のみレコードとして記録する
  - 閾値の値はそれぞれ以下:
    - sleep: 1min
    - unfocused: 2min
    - idle: 3min
- 複数の状態が併存しうるが、優先順位は sleep > unfocused > idle の順とする

### 3-4. 作業記録閲覧

- VS Code Sidebar から専用のWeb Viewパネルを開く
- 表示内容:
  - その日のトータル作業時間
  - ワークスペース別作業時間
    - ファイル別作業時間
  - 日付切り替えボタン
- トータル作業時間の算出方式:
  - ファイル別作業時間の合計
- ファイル別作業時間の算定方式
  - ended_at - started_at - 非作業時間

### 3-5. 設定

- MVPでは、可変的な設定値は以下のものだけ実装する:
  - 作業記録閲覧画面でのタイムゾーン
    - デフォルト値は固定値とせず、実行時にシステム(OS)のタイムゾーンを判定して決定する
      - 判定には `Intl.DateTimeFormat().resolvedOptions().timeZone` を利用する
    - システムのタイムゾーンを判定できない場合は `UTC` をフォールバックとして使用する
    - 設定値はIANAタイムゾーン名(例: `Asia/Tokyo`、`UTC`)で保持する
- 将来的に設定可能とする候補:
  - 非作業時間の閾値
  - SQLite DBファイルのパス

## スコープ外

- MVPからは除外し、将来的な機能追加を検討している機能:
  - 計測状態のアクティビティバー表示
  - WBS型タスク管理機能（特定のタスクにかかった時間の計測等）
