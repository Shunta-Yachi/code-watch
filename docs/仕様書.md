# 仕様書

## 概要

一日当たりのVS Code上の作業時間を記録するVS Code拡張機能(VSCE)

## 目的

- 一日当たりの作業時間を記録することで、作業履歴の振り返りや進捗管理などを可能にする
- ファイルごとの作業時間も併せて記録することで、その日特に時間コストがかかった作業・難航している作業等を可視化する

## 想定ユーザー

- 主な利用者は個人開発者とする
- ただし、将来的にはVS Code拡張機能として公開することを想定する
- 記録データはローカルに保存し、外部サービスへ送信しない

## MVPの範囲

MVPでは、以下の機能を実装対象とする。

- VS Codeウィンドウを開いたときの自動計測開始
- VS Codeウィンドウを閉じたときの自動保存
- 計測中の定期保存
- ワークスペースごとの作業時間記録
- アクティブエディタとして開いていたファイル等の時間記録
- 日別一覧のWebview表示
- 日別詳細のWebview表示
- 記録中のステータスバー表示
- JSON形式でのローカル保存
- タイムゾーン、保存先、放置判定閾値、ステータスバー表示の設定

## 計測の基本方針

### 自動開始・自動保存

- 拡張機能が有効なVS Codeウィンドウでは、ユーザー操作なしで計測を開始する
- VS Code起動後に拡張機能がactivateされたタイミングで、そのウィンドウの計測セッションを開始する
- VS Codeウィンドウを閉じる、VS Codeを終了する、拡張機能が無効化される等のタイミングで最終保存を行う
- 終了時保存だけに依存せず、計測中も定期的にJSONへ保存する
- 定期保存の目的は、VS Codeのクラッシュ、強制終了、OSスリープ等による記録欠損を減らすことである

### 時間カテゴリ

記録・表示の基本カテゴリは以下の3種類とする。

| カテゴリ   | 説明                                            |
| ---------- | ----------------------------------------------- |
| 総計測時間 | 計測セッションの終了時刻と開始時刻の差          |
| 実作業時間 | VS Codeがアクティブで、かつ放置状態ではない時間 |
| 非作業時間 | 総計測時間のうち、実作業時間ではない時間        |

総計測時間は、計測セッションの開始時刻と終了時刻から算出する。

```text
総計測時間 = 計測終了時刻 - 計測開始時刻
```

非作業時間は、総計測時間と実作業時間から算出する。

```text
非作業時間 = 総計測時間 - 実作業時間
```

ただし、総計測時間の算出元はカテゴリ別時間の加算結果ではなく、セッションの開始時刻と終了時刻とする。
実作業時間と非作業時間の合計は、原則として総計測時間と整合するように記録・検証する。

プログラムの実行タイミング、タイマーの遅延、イベント処理の順序により、カテゴリ別時間の合計と総計測時間の間には数ミリ秒程度の差分が発生し得る。
このため、完全一致は求めず、実装上は許容誤差を設けて検証する。
表示時にはミリ秒単位の差分をそのまま見せず、秒以上の単位に丸めて表示する。

### 実作業時間

- 実作業時間は、VS Codeウィンドウがアクティブで、放置状態ではない時間として記録する
- アクティブエディタとして開いているファイル等に対して、実作業時間を紐づける
- ファイル等の内訳では、アクティブエディタとして開かれていた時間を基準にする

### 非作業時間

- 非作業時間は、実作業時間ではない計測時間として記録する
- 非作業時間は計測を停止する理由にはしない
- 非作業時間には、可能な範囲で `nonActiveReason` を付与する
- `nonActiveReason` を保持することで、Webview側では非作業時間の合計を表示しつつ、必要に応じて理由別内訳も表示できる

`nonActiveReason` は以下の値を持つ。

| 値         | 説明                                                                                     |
| ---------- | ---------------------------------------------------------------------------------------- |
| `idle`     | VS Codeはアクティブだが、一定時間操作がない時間                                          |
| `inactive` | VS Codeウィンドウがバックグラウンドにある時間                                            |
| `sleep`    | タイマーの実行間隔が大きく空き、PCスリープ等により計測処理が停止していたと推定される時間 |

### 非作業理由の判定

- 放置判定の閾値は設定可能とする
- デフォルト値は5分とする
- VS Codeがアクティブな状態で、最後の操作から閾値以上経過した場合、`nonActiveReason` を `idle` として記録する
- VS Codeウィンドウがフォーカスを失っている場合、`nonActiveReason` を `inactive` として記録する
- PCスリープ判定には、放置判定と同じ閾値を用いる
- 定期計測タイマーの前回実行時刻から現在時刻までの差が閾値を超えた場合、その差分を `nonActiveReason: "sleep"` として記録する
- スリープ復帰後は、新しい計測区間として記録を継続する

## 日付とタイムゾーン

- 記録データ自体はUTCのタイムスタンプとして保存する
- 設定されたタイムゾーンは、日別集計やWebview表示にのみ使用する
- タイムゾーンのデフォルト値はUTCとする
- タイムゾーン設定には、`UTC`、`Asia/Tokyo` などのIANA Time Zone IDを使用する
- タイムゾーン設定を変更した場合、保存済みデータは変更しない
- タイムゾーン設定を変更した場合、表示上の日別集計だけが切り替わる

### 日付変更時の扱い

- 計測中に日付境界をまたいだ場合、設定されたタイムゾーン上の日付境界で区間を分割して集計する
- 例: タイムゾーンがUTCの場合、`2026-06-17T23:59:50Z` から `2026-06-18T00:00:10Z` までの実作業時間は、2026-06-17に10秒、2026-06-18に10秒として集計する
- 総計測時間も同様に、セッションの開始時刻と終了時刻の区間を日付境界で分割して集計する
- 非作業時間および `nonActiveReason` ごとの内訳についても同様に日付境界で分割して集計する

## ワークスペースの扱い

### ワークスペースごとの記録

- 記録はワークスペースごとに集計できる形で保存する
- Webviewの詳細画面でも、ワークスペースごとの表記を行う
- 同一ワークスペースを複数のVS Codeウィンドウで開いている場合、それぞれのウィンドウの記録を同一ワークスペースの記録として合算する
- 複数ウィンドウは、それぞれ独立した計測セッションとして扱う

### ワークスペース識別子

- 表示名にはワークスペース名を使用する
- 内部識別子には、ワークスペースフォルダの絶対パス、または `.code-workspace` ファイルのパスをもとにしたハッシュ値を使用する
- 絶対パスそのものは、原則として保存データに含めない
- ワークスペースを開いていないウィンドウは、`<no-workspace>` として記録する
- 複数のワークスペースなしウィンドウでの記録は、同じ `<no-workspace>` としてまとめる

### マルチルートワークスペース

- マルチルートワークスペースでは、ファイル等の表記にルートフォルダ名を含める
- 表記形式は以下を基本とする

```text
workspaceName/folderName/relative/path.ts
```

## ファイル等の記録対象

### 基本方針

- ファイルごとの作業時間は、アクティブエディタとして開いていた時間を基準にする
- 通常ファイル以外のエディタ、ビュー、パネルも、可能な範囲で記録対象とする
- 記録対象の表記は、ワークスペース名と相対パスを基本とする
- ファイル名のみ、または絶対パスのみでは保存しない

### 特殊な記録対象

通常ファイルとして表現できない対象は、以下のような特殊IDで記録する。

| 対象                   | 記録上の表記例                                 |
| ---------------------- | ---------------------------------------------- |
| Untitled               | `Untitled-1`                                   |
| 設定画面               | `<settings>`                                   |
| diffエディタ           | `<diff>`                                       |
| notebook               | `workspaceName/folderName/relative/path.ipynb` |
| ターミナル             | `<terminal>`                                   |
| 出力パネル             | `<output>`                                     |
| その他識別できない対象 | `<unknown>`                                    |

## 複数ウィンドウの扱い

- 各VS Codeウィンドウは、独立した計測セッションIDを持つ
- 各ウィンドウの計測結果は、ワークスペース識別子をもとに集計する
- 同一ワークスペースの複数ウィンドウで記録された時間は、ワークスペース単位で合算する
- MVPでは、同一時刻に複数ウィンドウで記録された時間の重複排除は行わない
- 通常は、実作業時間の条件がVS Codeウィンドウのアクティブ状態に依存するため、複数ウィンドウの実作業時間が大きく二重計上されることは想定しない
- 重複排除が必要になった場合は、将来的に集計時の時刻区間マージを検討する

## 保存仕様

### 保存先

- 保存先は設定可能とする
- デフォルトの保存先は、VS Code拡張機能のglobal storageとする
- 保存形式はJSONとする
- カスタム保存先が設定されている場合は、その保存先にJSONファイルを保存する

### 保存タイミング

- 計測中は定期的に保存する
- VS Codeウィンドウを閉じる、VS Codeを終了する、拡張機能が無効化される等のタイミングで最終保存を行う
- 複数ウィンドウが同時に記録する可能性があるため、保存時はウィンドウごとのセッションデータを失わないようにマージする

### 保存データの考え方

- 表示時のタイムゾーン変更に対応するため、日別集計済みデータだけを保存しない
- UTCの開始時刻・終了時刻を持つ計測区間として保存する
- Webview表示時に、設定されたタイムゾーンで日別集計を行う

保存データは、概念的には以下の情報を持つ。

```json
{
  "version": 1,
  "sessions": [
    {
      "sessionId": "window-session-id",
      "workspaceId": "hashed-workspace-id",
      "workspaceName": "workspace-name",
      "startedAt": "2026-06-17T00:00:00.000Z",
      "endedAt": "2026-06-17T01:00:00.000Z",
      "segments": [
        {
          "startedAt": "2026-06-17T00:00:00.000Z",
          "endedAt": "2026-06-17T00:10:00.000Z",
          "category": "active",
          "resource": "workspaceName/src/extension.ts"
        },
        {
          "startedAt": "2026-06-17T00:10:00.000Z",
          "endedAt": "2026-06-17T00:15:00.000Z",
          "category": "nonActive",
          "nonActiveReason": "idle",
          "resource": "workspaceName/src/extension.ts"
        }
      ]
    }
  ]
}
```

`category` には、少なくとも以下を保存する。

| 値          | 意味       |
| ----------- | ---------- |
| `active`    | 実作業時間 |
| `nonActive` | 非作業時間 |

`category` が `nonActive` の場合、`nonActiveReason` として以下を保存する。

| 値         | 意味                                             |
| ---------- | ------------------------------------------------ |
| `idle`     | VS Codeはアクティブだが、一定時間操作がない時間  |
| `inactive` | VS Codeウィンドウがバックグラウンドにある時間    |
| `sleep`    | PCスリープ等により計測処理が停止していた推定時間 |

## Webview仕様

### 表示場所

- Activity Barに専用アイコンを追加する
- サイドバーから専用Webviewを表示する

### 日別一覧

MVPの日別一覧では、以下の列のみ表示する。

| 列         | 内容                               |
| ---------- | ---------------------------------- |
| 日付       | 設定されたタイムゾーンに基づく日付 |
| 実作業時間 | その日の実作業時間の合計           |

- 今日の記録も日別一覧から確認する
- 日付行を選択すると、日別詳細を表示する

### 日別詳細

日別詳細では、選択した日付について以下を表示する。

- 総計測時間
- 実作業時間
- 非作業時間
- 非作業時間の理由別内訳
- ワークスペースごとの内訳
- ファイル等ごとの内訳

非作業時間の理由別内訳では、`idle`、`inactive`、`sleep` ごとの時間を表示する。

ファイル等ごとの内訳では、以下の列を表示する。

| 列         | 内容                                                     |
| ---------- | -------------------------------------------------------- |
| ファイル等 | `workspaceName/folderName/relative/path.ts` または特殊ID |
| 実作業時間 | アクティブエディタとして開かれていた実作業時間           |
| 非作業時間 | その対象に紐づく非作業時間                               |
| 総計測時間 | その対象に紐づく計測時間の合計                           |

## ステータスバー仕様

- 記録中はステータスバーにカウントを表示する
- 表示する値は、設定されたタイムゾーンにおける当日の実作業時間とする
- ステータスバー表示は設定で有効・無効を切り替えられる

## 設定項目

MVPでは、以下の設定項目を提供する。

| 設定               | デフォルト値   | 説明                                             |
| ------------------ | -------------- | ------------------------------------------------ |
| タイムゾーン       | `UTC`          | 日別集計とWebview表示に使用するIANA Time Zone ID |
| 保存先             | global storage | JSONファイルの保存先                             |
| 放置判定閾値       | 5分            | `idle` および `sleep` の判定に使用する           |
| ステータスバー表示 | 有効           | 記録中のステータスバー表示を切り替える           |
