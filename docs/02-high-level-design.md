# High-Level Design Document

## 1. 技術スタック

- TypeScript
- SQLite

## 2. 作業時間記録

- 計測した作業時間を記録する
- 記録先はSQLiteを採用
  - 理由: 1ファイル完結の手軽さ、競合処理
  - テーブル設計は `docs/03-low-level-design.md` を参照
- 記録タイミング:
  - 1: 計測が終了したとき
  - 2: 計測中、定期的に保存(ハートビート)
    - デフォルト1分毎
- 複数ワークスペースが開かれている場合、同数のセッションを並行稼働させる

## 3. 非作業時間記録

- ただし、これらはすべてDBに保存するとレコード数が肥大化し破綻する(特にidle)
  - そのため、一定の閾値を拡張機能側で設定し、その閾値を超えた場合のみレコードとして記録する
  - 閾値の値はそれぞれ以下:
    - sleep: 1min
    - unfocused: 2min
    - idle: 3min
- 複数の状態が併存しうるが、優先順位は sleep > unfocused > idle の順とする

## 4. 作業記録閲覧

- トータル作業時間の算出方式:
  - ファイル別作業時間の合計
- ファイル別作業時間の算定方式
  - ended_at - started_at - 非作業時間

## 5. 設定

- MVPでは、可変的な設定値は以下のものだけ実装する:
  - 作業記録閲覧画面でのタイムゾーン
    - デフォルト値は固定値とせず、実行時にシステム(OS)のタイムゾーンを判定して決定する
      - 判定には `Intl.DateTimeFormat().resolvedOptions().timeZone` を利用する
    - システムのタイムゾーンを判定できない場合は `UTC` をフォールバックとして使用する
    - 設定値はIANAタイムゾーン名(例: `Asia/Tokyo`、`UTC`)で保持する
- 将来的に設定可能とする候補:
  - 非作業時間の閾値
  - SQLite DBファイルのパス
